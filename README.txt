ExifTool Folder Reporter v1.4.0
================================

Purpose
-------
This is a browser tool for student exercises. Students select a folder and generate:

- tree.txt
- file-manifest.csv
- exiftool-metadata.json
- exiftool-metadata.csv
- exiftool-report.txt
- exiftool-processing-issues.csv
- one ZIP bundle containing the above files

Files are processed by the browser. The tool does not upload files to any server.

Filtering note
--------------
The app ignores macOS .DS_Store files at folder-selection time, so they are excluded from ExifTool processing, tree output, the file manifest and all reports.

Packaging note
--------------
v1.2.0 and later keep the WebAssembly runtime in zeroperl.wasm instead of inlining it into app.js. This makes app.js much smaller and easier to inspect. Keep index.html, app.js, style.css and zeroperl.wasm in the same folder.

Student instructions
--------------------
1. Open the hosted page, or run this folder through a small local web server.
2. Click "Choose exercise folder" and select the folder supplied for the activity.
3. Click "Run ExifTool report".
4. Click "Download all reports as ZIP" and submit the ZIP file if instructed.

Local-server fallback
---------------------
Most browsers will not load a separate WebAssembly file reliably from a file:// page. If opening index.html directly does not work, run a tiny local web server from this folder.

Option A: Python installed
- macOS: open Terminal in this folder and run: python3 -m http.server 8000
- Windows: open PowerShell in this folder and run: py -m http.server 8000
- Then open: http://localhost:8000

Option B: VS Code installed
- Open this folder in VS Code.
- Use the Live Server extension, if available.

Teacher notes
-------------
The app uses @uswriting/exiftool, which packages ExifTool 13.42 via WebAssembly. The browser folder picker uses the webkitdirectory file-input capability and File.webkitRelativePath values to reconstruct the directory tree.

ExifTool arguments
------------------
-csv

The only argument supplied to ExifTool is -csv. Browser folder selection supplies the recursive file list that command-line ExifTool would normally obtain from -r.

Limitations
-----------
- The browser only sees files selected by the student.
- Very large folders can be slow. Keep exercise folders modest.
- The app reads metadata and generates reports only; it does not write or remove metadata.
- The merged metadata CSV is generated by combining ExifTool -csv output from each processed file. SourceFile, Directory and FileName are restored from the browser-provided relative path to approximate exiftool -csv -r DIR output. Files that error or are skipped are reported separately in exiftool-processing-issues.csv so the metadata CSV remains ExifTool-style.
