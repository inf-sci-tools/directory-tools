ExifTool Folder Reporter v1.2.0
===============================

Purpose
-------
This is a local browser tool for student exercises. Students select a folder and generate:

- tree.txt
- file-manifest.csv
- exiftool-metadata.json
- exiftool-metadata.csv
- exiftool-report.txt
- one ZIP bundle containing the above files

Files are processed by the browser. The tool does not upload files to any server. Chrome/Firefox may use the word "upload" in the folder-permission dialog, but in this local tool it means "make the selected local files available to this web page".

Packaging change in v1.2.0: zeroperl.wasm is kept as a separate runtime asset instead of being inlined into app.js. This keeps app.js small and inspectable. Keep index.html, app.js, style.css and zeroperl.wasm in the same folder.

Fix retained from v1.1.1: resolves a folder-selection crash that could show "Cannot read properties of undefined (reading 'replaceAll')" after selecting a directory.

Recommended student instructions
--------------------------------
1. Unzip this distribution ZIP, or use the hosted GitHub Pages version.
2. Open index.html from a lightweight local web server, or open the hosted page.
3. Confirm that the status box says: "Ready. JavaScript loaded. Select a folder to begin."
4. Click "Choose exercise folder" and select the folder supplied for the activity.
5. Confirm that the status box says how many files the browser made available.
6. Click "Run ExifTool report".
7. Click "Download all reports as ZIP" and submit the ZIP file if instructed.

If the page still says JavaScript has not started
-------------------------------------------------
You are probably opening a non-runnable source copy, or the browser has blocked/missed app.js.
Make sure index.html, app.js, style.css and zeroperl.wasm are all in the same folder.

Fallback if double-clicking the HTML does not work
--------------------------------------------------
Most browsers will not load a separate WebAssembly file reliably from file:// pages. Run a tiny local web server from this folder.

Option A: Python installed
- macOS: open Terminal in this folder and run: python3 -m http.server 8000
- Windows: open PowerShell in this folder and run: py -m http.server 8000
- Then open: http://localhost:8000

Option B: VS Code installed
- Open this folder in VS Code.
- Use the Live Server extension, if available.

Teacher notes
-------------
The app uses @uswriting/exiftool, which packages ExifTool 13.42 via WebAssembly. The browser folder picker uses the webkitdirectory file-input capability and File.webkitRelativePath values to reconstruct the directory tree.

Default ExifTool arguments
--------------------------
-json -a -u -G1 -s -n

These include duplicate tags, unknown tags, group names, raw tag names and numeric values.

Limitations
-----------
- The browser only sees files selected by the student.
- Very large folders can be slow. Keep exercise folders modest.
- The app reads metadata and generates reports only; it does not write or remove metadata.
- ExifTool CSV output is generated by the app from per-file JSON records rather than by one recursive ExifTool command. This avoids command-line and local filesystem permission issues in the browser.
